## Коротко

Я студент ИТМО, интересуюсь веб‑разработкой.

## Навыки

- **Frontend:** HTML/CSS, JavaScript/TypeScript.
- **Инструменты:** Git, GitHub Actions, CI/CD, базовые практики тестирования

## Опыт и учебные проекты

Опыта мало, но я стараюсь.

### 1. Подход к разработке

Я стараюсь держать проект в состоянии, когда его можно:

- быстро собрать локально;
- повторяемо задеплоить;
- легко расширять за счёт понятной структуры.

### 2. Что считаю важным

**Читаемость и поддерживаемость** важнее “магии”. Я предпочитаю:

- явные конфиги;
- короткие скрипты сборки;


### 3. Текст

Hugo хорош тем, что позволяет хранить контент рядом с исходниками и деплоить его как статический сайт. Для портфолио это удобно: страницы быстро грузятся, контент редактируется в Markdown, а оформление можно менять темой.

#### Список тезисов

1. Контент — в `content/`.
2. Конфиг — `hugo.toml`.
3. Статика — `static/`.
4. Деплой — GitHub Actions.

#### Ещееееее длинный текст

Статические генераторы сайтов подходят, когда нужно:

- публиковать материалы без сложного бекенда;
- иметь контроль над структурой и версиями;
- получать быстрый сайт и простую инфраструктуру.

Если нужно больше динамики, обычно используют внешний API или отдельный сервис, но для портфолио (и учебных задач) статической публикации чаще всего достаточно.
